# Wails3 Cockpit - Script Bridge Architecture Plan

Date: 2026-01-21

## Overview
Wails3 desktop app with "cockpit" UI showing tiles for host processes. Bun/Python scripts can be started from UI or CLI, with real-time monitoring via network bridge.

## Architecture Decision

### Communication Pattern: HTTP + SSE
- HTTP endpoints for script commands/registration
- Server-Sent Events for real-time log streaming
- No WebSocket library needed (browser native SSE + Go stdlib)

### Script Protocol Types
1. **NETWORK_BRIDGE**: Script calls home (POST /api/script/heartbeat, POST /api/script/log)
2. **STDOUT_POLL**: Script outputs JSON lines to stdout, go layer captures
3. **FILE_WATCH**: Script writes to designated log file, go layer tails

## Key Files (Stubs)

### `/cockpit/app/process_manager.go`
- `ProcessManager` struct handling script lifecycle
- `ScriptInstance` type for tracking running scripts
- `StartScript()` launches scripts, captures stdout/stderr
- HTTP handlers: `/api/script/register`, `/api/script/heartbeat`, `/api/script/log`
- SSE endpoint `/api/stream` for frontend subscription

### `/cockpit/app/script_catalog.go`
- TOML catalogue schema for script metadata
- `LoadCatalog()`, `BuildRegistry()` functions

### `/cockpit/app/scripts/templates/bridge-client.ts`
- `ScriptBridge` class for bun scripts to call home
- Methods: `register()`, `heartbeat()`, `log()`, start/stop heartbeat

### `/cockpit/app/frontend/src/components/script-tile.ts`
- `ScriptTile` Lit component for displaying script status/logs
- Subscribes to SSE for real-time updates
- Start/Stop controls

### `/cockpit/app/frontend/src/lib/sse-client.ts`
- Minimal SSE client (no external deps)

### `/cockpit/app/scripts/generate.ts`
- Bun CLI tool to generate scripts with bridge integration

## Catalogue Schema (TOML)
```toml
[[script]]
id = "backup-db"
name = "Database Backup"
type = "bun"
entry = "scripts/backup/main.ts"
protocol = "stdout_poll"
run_mode = "oneshot"
tags = ["maintenance", "daily"]
```

## Design Principles
- Minimal 3rd party deps (browser native SSE, Go stdlib)
- Go layer is intermediary only (no business logic)
- Single source of truth: TOML catalogue
- CLI scripts auto-appear in UI via network bridge
